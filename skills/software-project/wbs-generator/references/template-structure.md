## 列定义 (Row 8)

| 列 | 表头 | 说明 |
|---|------|------|
| A | 任务 | 任务名称，分区标题行仅有A列和D列 |
| B | 负责人 | 格式: `@角色-姓名`，多人用`@`分隔 |
| C | 状态 | 下拉: 待启动/进行中/已完成/取消 |
| D | 预计结束时间 | 格式: `YYYY.MM.DD` |
| E | 备注 | 详细说明，常以`【要求】`开头 |
| F | 辅助信息 | 非任务列，放参考资料链接等 |

## 典型分区结构

以下为模板中常见的分区，不同项目可增删调整：

### 1. 里程碑与版本计划

全新项目的标准里程碑（按顺序，不可随意删减）：

1. **方案评估** — 编码性能+AI性能+DDR带宽评估，确认方案可行性
2. **首次样机版本** — 基本出图，核心驱动OK，基本功能可演示
3. **单元测试版本** — 各模块单元功能验证通过，主要功能可用
4. **满负载版本** — 所有码流+AI+网络同时满负载运行稳定
5. **整机测试版本** — 全功能集成，提交测试部进行系统测试
6. **DVT老化版本** — 72小时以上拷机无异常，内存无泄漏，功耗温升达标
7. **PVT贴片版本** — PVT贴片板验证，硬件量产一致性确认
8. **PVT正式版本** — 量产正式版本，所有问题闭环
9. **专项测试** — 针对项目特点的专项测试（如PT/AF/WiFi等）
10. **转测** — 转测试部全面测试，提供转测文档
11. **版本发布** — 正式发布量产版本

功能升级/迭代项目可根据实际情况裁剪（如跳过方案评估、合并PVT阶段等）。

### 2. 变更记录
列：变更记录(A)、干系人(B)、变更时间(D)、影响范围(E)

### 3. 产品
产品规格定义、编码规格等。

### 4. 媒体驱动
sensor驱动、性能评估、WiFi模组驱动等。

### 5. 图像相关
FOV、热噪、横纹等图像质量任务。

### 6. 业务适配
工程打包、灯光适配、WiFi/蓝牙适配、资源评估等。

### 7. 新增业务适配
新功能开发，任务名通常以`【新】`前缀标记。

### 8. 研发和测试
checklist、兼容性测试、烤机测试、场景测试、专项测试等。

### 9. 风险、关键路径、重点问题
已知风险和待解决重大问题。

## 格式特点

- 分区标题行：A列为分区名，D列为"预计结束时间"，其余列为空，无边框
- 分区标题行与上方内容之间留1行空行
- 任务行有完整的A-E列填充
- 状态默认为"待启动"
- Row 9 为提示行: B9="输入"@+人名"提及负责人", C9="使用下拉列表显示", E9=提示文本
